### 🚀 Deployment Strategy: Render × Vercel

One of my biggest technical challenges was solving cold start latency when deploying a MERN stack e-commerce site.
To improve the user experience, I chose to **split the deployment**:

- **Front-end on Vercel** for faster loading and SSR

- **Back-end API on Render**

This approach significantly improved performance and loading time.
You can even compare both versions:

- **Athleisure Styles**

  - 🔗 [Slow version (Render only)](https://athleisurestyles.onrender.com)

  - 🔗 [Recommended version (Vercel + Render)](https://mern-athleisure-styles.vercel.app/)

## What is this project?

**(EN)**

`"react-myPortfolio"` is a **fully responsive, bilingual portfolio website** built with **React and Tailwind CSS**, enhanced by **Framer Motion** for rich, engaging animations. It was designed with a strong focus on user experience, ensuring seamless performance across devices and languages.

The site features **dark/light mode switching, English/Japanese language support**, and clean, intuitive layouts tailored through countless iterations. From font sizing to animation behavior, every element was refined through hands-on testing to provide a comfortable and visually appealing user experience.

One of the key highlights is the **Projects section**(titled 'Portfolio' on the site), where I showcase a wide range of development styles — from **fully deployed full-stack applications** to **front-end focused projects**, some of which include custom Webpack-based environments.

I’ve also explored backend development with projects such as a **Twitter-style message board using PHP & MySQL**, and a **Tinder-style matching app built with Laravel**. While some of these backend projects haven’t been deployed due to infrastructure constraints, **all code is publicly available on GitHub**, demonstrating the thought process and technical depth behind each build.

Through this portfolio, I hope you'll get a clear sense of my eye for UI/UX, **my commitment to building with empathy**, and my **willingness to dive into new technologies beyond front-end development**.

## 📖 Installation

### 🚧 Prerequisites

To run this project locally, you’ll need:

- [Node.js](https://nodejs.org/en) v20.x (Tested with v20.11.1)
- [npm](https://www.npmjs.com/) v10.x (comes with Node.js)
- [Git](https://git-scm.com/) – for cloning the repository
- [React](https://react.dev/) v18+ (installed automatically via `npm install`)

> ✅ This project has been tested with **Node.js v20.11.1**. Other recent versions may also work.
> <br />

### 📦 Key Dependencies

This portfolio project was built with:

- [React](https://create-react-app.dev/docs/getting-started/) (via Create React App)

- [Tailwind CSS](https://tailwindcss.com/) - for utility-first styling

- [Framer Motion](https://motion.dev/) - for smooth animations

- [React Hook Form](https://react-hook-form.com) - for flexible form validation

- [Icon Scout](https://iconscout.com) - for icon assets
  <br />

## Usage 🚀

#### 1. Clone this repository

```bash
git clone https://github.com/sahoooii/react-myPortfolio.git
```

#### 2. Navigate to the project directory

```bash
cd react-myPortfolio
```

#### 3. Install dependencies

```bash
npm install
```

#### 4. Set up environment variables

Rename the `example.env` file to `.env` and add the following

```bash
REACT_APP_FORM_SUBMIT=ADD_YOUR_GET_FORM_INFO
REACT_APP_MY_EMAIL=ADD_YOUR_OWN_EMAIL
```

#### 5. Start the development server

```bash
npm start
```
